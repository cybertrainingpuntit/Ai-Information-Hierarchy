# AI Information Hierarchy – Cybertraining EU B.V.

## What is this?
This repository contains a **lightweight folder‑and‑file structure** that stores all the core context your AI assistants need to work for **Cybertraining EU B.V.** (website: https://cybertraining.it, platform: https://cyberpulse.it).

It follows Rick Mulready’s *AI Information Hierarchy* approach:
1. One fixed hierarchy lives on disk (or in a repo).
2. Four core business files (`About Me.md`, `About My Business.md`, `About My Voice.md`, `My Offers.md`).
3. Project templates (e.g. `CyberPulse Platform`) with sub‑folders `Instructions`, `Voice`, `References`, `Examples`, `Notes`.
4. Prompt templates you can paste into Claude/ChatGPT to fill the files automatically.

Having this structure means you never have to repeat the same background information for every new AI chat – you simply point the model at the folder and it instantly knows who you are, what you sell, and how you sound.

## How to use it
### 1️⃣ Initialise the hierarchy
```bash
# From the repository root (or any folder you want the hierarchy in)
python3 setup_hierarchy.py
```
The script will:
- Create a `My Business/` folder with the four markdown files containing placeholder headings.
- Create a sample project folder `CyberPulse Platform/` with the five sub‑folders and an empty `Instructions/Instructions.md`.
- Add a minimal `.gitignore`.

You can also specify a custom base path:
```bash
python3 setup_hierarchy.py /path/to/your/hierarchy
```

### 2️⃣ Fill the business files (Interview‑style Prompt 2)
Open `prompts/prompt_2_fill_business.md` and copy‑paste its content into your AI assistant (Claude, GPT‑4, etc.). The AI will ask you a series of short questions (About Me, Business, Voice, Offers). Answer in your own words – the AI will write the markdown files for you.

### 3️⃣ Create project‑specific instructions (Prompt 3)
Open `prompts/prompt_3_project_instructions.md` and let the AI interview you about a concrete project (e.g. a Cyber‑Awareness newsletter). The result will be saved as `CyberPulse Platform/Instructions/Instructions.md`.

### 4️⃣ Version control (highly recommended)
```bash
git init
git add .
git commit -m "Initial AI hierarchy for Cybertraining EU B.V."
```
Every time you improve a file (e.g. update the Voice description or add a new Offer) commit the change – the AI will always read the latest version.

### 5️⃣ Query the hierarchy from an LLM
When you start a new chat, simply tell the model:
```
Read the files in the `My Business/` folder to understand my company (Cybertraining EU B.V.) and the `CyberPulse Platform/Instructions/Instructions.md` for this specific project.
```
Most modern LLMs accept a brief *"file‑upload"* or *"read from folder"* instruction; you can also copy‑paste the relevant markdown sections.
